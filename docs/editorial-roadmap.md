# מפת דרכים עריכתית

מסמך העבודה המרכזי לתוכן של מקור בדיקה. המטרה: לשמור על עקביות בין האתר, הקוד והדוקס בלי להפוך את הריפו לתוכנית שיווק.

## נקודת פתיחה ל-v0.3

נכון ל-23 ביוני 2026, המערכת היא אתר Next.js סטטי-תחילה עם תוכן בדיקות typed, loader משותף, route דינמי ויצירת metadata, sitemap ונתונים מובנים מתוך רשומות התוכן. יעד התוכן הראשון של 18 בדיקות הושלם, והמאגר מתקדם כעת ליעד תוכן הבא בלי לשנות את חוויית הקריאה הציבורית.

### הושלם ונחשב לבסיס קיים

- הקשחת runtime: כותרות אבטחה גלובליות, CSP תואם Vercel Analytics, גרסאות תלויות נעולות ובדיקת production build.
- שלמות תוכן ו-CI: בדיקות למיפוי עמודים, sitemap, Open Graph ואשכולות; `check:content` רץ כחלק מה-CI.
- מובייל: header מצומצם, ניווט תחתון, אייקונים עקביים, רוחבים ומרווחים מותאמים למסכים צרים.
- גילוי וסורקים: `robots.txt`, `sitemap.xml`, `llms.txt`, canonical metadata ותמונות שיתוף.
- חיפוש מקומי: חיפוש חי על הבדיקות שכבר פורסמו, ללא חיפוש AI פתוח.
- אשכולות נושא: נתוני אשכולות, עמודי אשכול דינמיים, מפת המשך וקישור בין בדיקות, תגיות ואשכולות.
- אמון ומטא-דאטה: WebSite, Organization ו-Person JSON-LD, אחריות עורך, מדיניות עריכה, תיקונים ומתודולוגיה.
- הצעות ושיתוף: טופס הצעה שפותח GitHub Issue מוכן מראש, העתקת תקציר ושיתוף WhatsApp.
- תצוגת ראיות: גלריה והגדלה, נכסי SVG תקינים ותמונות מקור שנשמרות בפורמט המקורי שלהן.

### מצב התוכן הנוכחי

- schema ו-workflow typed מוגדרים ב-`lib/content/claim-schema.ts`.
- 19 הבדיקות שפורסמו נשמרות כרשומות תחת `content/claims/`.
- `lib/content/claim-loader.ts` מספק API אחיד ל-routes, לחיפוש, לאשכולות ולצרכני SEO/GEO.
- עמודי הבדיקות מרונדרים דרך `app/claims/[slug]` עם `generateStaticParams`.
- metadata, sitemap, `llms.txt` ונתונים מובנים נוצרים מרשומות התוכן.
- route hygiene מכוסה ב-`lib/route-hygiene.test.ts`: נתיב XRP הישן מפנה לכתובת הקנונית, `/sermap` נשאר 404 ונתיבי topic ציבוריים תמיד concrete.
