# Domain migration checklist

סטטוס: לשימוש רק אם מחליטים לעבור מ־`ai-source-lab.vercel.app` לדומיין קבוע.

תאריך בדיקה משוער: סביב 2026-07-08, בתום חלון 14 ימי הניסוי/ייצוב הנוכחי.

התאריך הוא checkpoint בלבד. הוא לא מחייב קניית דומיין, שינוי שם או שינוי עיצוב.

## כלל החלטה

לא קונים דומיין כדי להרגיע לחץ.

קונים דומיין רק אם מתקיים אחד מאלה:

1. עברו 14 ימי ניסוי ויש סימני חיים לפרויקט.
2. קיימת בעיית אינדוקס אמיתית ומתמשכת גם אחרי בדיקות טכניות.
3. מתקבלת החלטה מיתוגית שהפרויקט צריך כתובת קבועה.

## למה לא למהר

מעבר דומיין הוא מעבר URL.

זה אומר שצריך להתייחס אליו כמו שינוי כתובות ולא כמו החלפת לוגו. בזמן המעבר ייתכנו תנודות בסריקה, בדוחות ובתוצאות, ולכן לא מערבבים אותו עם שינויים גדולים אחרים.

## לפני מעבר

- [ ] לבחור דומיין קצר, ברור וזול מספיק.
- [ ] לוודא שכל הכתובות הקיימות ממופות לכתובות החדשות.
- [ ] להכין Redirects קבועים מהכתובות הישנות לחדשות.
- [ ] לוודא שה־canonical בכל עמוד יעבור לדומיין החדש.
- [ ] לעדכן `siteUrl`.
- [ ] לעדכן sitemap.
- [ ] לעדכן robots אם צריך.
- [ ] לעדכן Open Graph URLs.
- [ ] לעדכן Bing Webmaster ו־Google Search Console.
- [ ] לא לפרסם במקביל שינוי עיצוב גדול או שינוי מבנה תוכן גדול.

## אחרי מעבר

- [ ] לבדוק שהדומיין הישן מפנה לדומיין החדש.
- [ ] לבדוק כמה עמודים מרכזיים ידנית.
- [ ] להגיש sitemap חדש.
- [ ] לעקוב אחרי GSC/Bing במשך כמה ימים.
- [ ] לא להיבהל מתנודות זמניות.

## כלל זהירות

אם אין תקלה אמיתית, מחכים ליום 14 לפני החלטה.
